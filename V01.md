# V01 Project Memory

> Stable project memory loaded at the top of every agent context. Keep it concise and factual.

## Stack
- Language: Python 3.x
- ML: scikit-learn (Multinomial Naive Bayes + TF-IDF)
- Data: pandas
- Plotting: matplotlib
- Numerics: numpy
- API: FastAPI + Uvicorn
- Validation: Pydantic v2
- Persistence: SQLAlchemy 2.x (SQLite by default, configurable via `DATABASE_URL`)
- Realtime: WebSockets via FastAPI + in-process asyncio pub/sub bus
- Serialization: joblib for fitted pipeline artifacts

## Environment
- Package manager: pip (Python)
- OS: windows
- Git: no

## Build & Run
- install: `pip install -r requirements.txt`
- train + evaluate (CLI): `python train_spam_classifier.py`
- run API server: `uvicorn app:app --host 0.0.0.0 --port 8000` (or `python app.py`)
- CLI output: prints accuracy/precision/recall/F1 + classification report to stdout, saves confusion matrix to `artifacts/confusion_matrix.png`
- API output: persists model + run records to `artifacts/spam_classifier.db` and fitted pipeline to `artifacts/model_<ts>.joblib`

## Conventions
- Single-file training script (`train_spam_classifier.py`) keeps the pipeline reproducible.
- Reusable training logic lives in `pipeline.py` (imported by both the CLI and the API).
- Dataset is read from `spam.csv` in the project root (latin-1 encoded SMS Spam Collection).
- Labels: `ham` -> 0, `spam` -> 1.
- Train/test split is stratified (80/20, `random_state=42`).
- API request/response contracts are defined in `schemas.py` (Pydantic v2).
- ORM models and repositories live in `db.py`; `init_db()` is idempotent and runs on app startup.

## Architecture Notes
- Pipeline: `TfidfVectorizer(ngram_range=(1,2), min_df=2, sublinear_tf=True)` -> `MultinomialNB(alpha=0.1)`.
- Metrics reported: accuracy, precision, recall, F1 (positive class = spam), plus full `classification_report`.
- Confusion matrix is rendered with `sklearn.metrics.ConfusionMatrixDisplay` and saved via matplotlib.

### Phase 2 — Backend Core
- **Service layout**: `app.py` (FastAPI routes + lifespan + CORS + WS) → `service.py` (orchestration) → `pipeline.py` (training/inference) → `db.py` (SQLAlchemy ORM + repositories).
- **Domain & persistence**: `schemas.py` defines Pydantic v2 request/response contracts (`PipelineConfigIn`, `ModelOut`, `TrainRequest`, `PredictRequest`, `RunOut`, event payloads). `db.py` defines ORM models `ModelRecord` and `RunRecord` with JSON columns for config + class distribution, plus `ModelRepository` and `RunRepository` for CRUD. `init_db()` is idempotent and runs on app startup. Engine URL is configurable via `DATABASE_URL` (defaults to SQLite at `artifacts/spam_classifier.db`).
- **Realtime**: `events.py` exposes an in-process `EventBus` (asyncio queues with a lock-guarded subscriber set). `service.py` publishes `progress`, `run.completed`, and `model.created` events; `app.py` fans them out to WebSocket subscribers on `/ws/events` after sending a `hello` frame.
- **Background training**: `POST /models/train` returns `202 Accepted` with `run_id` + `model_id`; the actual training runs in `loop.run_in_executor` so the event loop stays responsive and progress events stream live. Progress callbacks persist to the DB and publish events via `asyncio.run_coroutine_threadsafe` from the worker thread.
- **Inference**: `POST /predict` accepts `texts` and an optional `model_id` (defaults to most recent). Loads the joblib artifact and returns per-text `label` + `spam_probability`.
- **API surface**:
  - `GET  /health`
  - `POST /models/train` (202)
  - `GET  /models`, `GET /models/{model_id}`
  - `POST /predict`
  - `GET  /runs`, `GET /runs/{run_id}`
  - `WS   /ws/events` (hello → progress → run.completed → model.created)
